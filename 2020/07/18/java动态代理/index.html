java动态代理

发表于2020-07-18|更新于2022-12-09|面经
|阅读量:

真是对象不能直接访问，需要通过代理对象才可以访问。

Proxy类似一个“调度器”，InvocationHandler相当于一个增强器，提供一些额外的服务。

小结

一个接口，真是对象。
一个真实对象(被代理对象)。
一个通过Proxy.newProxyInstance()产生的代理类(实现InvocationHandler接口，并实现invoke()，invoke()方法内可以对真实对象的方法进行前置增强和后置增强；通过Proxy.newProxyInstance(ClassLoader, interfaces, InvocationHandler)创建代理对象，该代理对象需要实现Girl接口)。

静态代理没有实现InvocationHandler接口，创建对象时被代理对象的业务发生变化时，代理对象也要改(违反了开闭原则，对修改关闭，对扩展开放)，导致可维护性差，可扩展能力差。动态代理可以解决这个问题。使用动态代理后，接口中的方法的增加或减少，并不影响代理类中的代码，因为他是通过反射进行方法调用的。

使用动态代理代理多个对象时，只需要定义其接口和真实对象(被代理对象)的类即可，通过同一个代理类进行前置和后置增强，只不过前置增强和后置增强的方法是一样的。

代理对象的使用：
新建真实对象，并将真实对象传递到代理类的构造函数中，通过代理类创建代理对象，将代理对象强转成接口，然后调用。

业务扩展：代理对象代理多个真实对象。
