552.学生出勤记录II

发表于2020-04-21|更新于2022-06-09|动态规划
|阅读量:

题目描述

分析

首先搞清楚题目中所说的“超过两个连续的L”是什么意思。他是说像这样的序列：LLLAPPPL，PPPALLLLP。而LLPLLPLLPLL是可被奖励的出勤序列。

因为n值最大为十万，所以算法的时间复杂度只能是O(n)才可通过，否则超时。

这道题不是求最大最小值，也不是计数问题，但仍可以使用动态规划。

首先定义状态：
直觉中，如果有了状态的话，那么d(状态a)=可被奖励的次数。因为结果就是求的可被奖励的次数嘛。
所以，就接着想到什么可影响到可被奖励的次数呢？显然，有题意知：出勤序列中少于等于一个A，且没有出现连续三个或三个以上的L。既然如此，那状态中一定有一个维度记录出勤序列中A出现的次数，以及连续出现的L的个数。有了这些还不够，试想想用动态规划，这题只能走地推式的方法，因为n是十万级别的递归会栈溢出。也就是说我们知道了n=i的结果，那该如何导出n=i+1的结果呢？分析至此，状态也就定义出来了：
d[len][i][j]：表示出勤序列的长度为len，且出现了i次A，且从当前位置(即len位置)连续出现了j次L。
如LLPPLLA可表示为d[7][1][2]，len是从后往前数的。
LPPPLLP可表示为d[7][0][1]。
PAP可表示为d[3][1][0]。

那base case该如何定义呢？

考虑一些特殊情况：当j>3时，d[len][i][j] = 0; 当i>1时，d[len][i][j] = 0; 这在遍历状态时，直接约定了其有用状态的可选空间。

当j = 0时，我们考虑如何递推求得d[len][i][0]。
根据状态的含义，d[len][i][0]限定了len位置只能是P或者A。考虑len位置是P，那么A肯定出现在P的后面。

当j = 1时，由状态的含义知len位置上是L，那么len-1位置上只能是A或者P，而不能是L，因为如果是L的话，j就要等于2了，这与假设不服。
故d[len]i][1]=d[len-1][i][0];

当j = 2时，由状态的含义只len位置与len-1位置是L，假设len > 2，len-3位置可以是A或者P，但不能是L。因此d[len][i][2]=d[len-2]][i][0]; 当len==2 && i+2<=len时，d[len][i][2] = 1;
