SS和SSR的过滤规则

发表于2020-04-01|更新于2022-06-09|others
|阅读量:
这篇文章讲述如何设计和维护自己的Adblock Plus过滤规则。这些过滤规则限制了你可以哪些站点可以访问，哪些有意要屏蔽的。可屏蔽的内容有广告，图片，requests一级scripts。

Adblock Plus filters

过滤名单(浏览器去阻止访问的一系列规则)包括两部分：

  • Adblock Plus预装的
    • Acceptable Ads
    • EasyList (+ bundled language filter list - depending on your browser’s language setting)
    • ABP Anti-circumvention filter list
  • 自定义的

filters的种类

  • Blocking filters

    用于拦截并阻塞请求

  • Content filters / element hiding filters

    用于隐藏网页中特殊的元素及snippets。

  • Exception filters

    将某些已定义的过滤规则排除出拦截范围之外。

基础过滤规则

1. 定义Exception rules

​ 用@@标识一个Exception rule。如果你所写的过滤规则中包含了你想unblock的内容，就使用Exception rule将它排除出过滤范围之内。Exception rule也可以使用通配符和正则表达式。

例如：
你不想拦截http://example.com/advice.html，则可以使用@@advice，将路径中含有advice的exclude掉。
若使用^document可以exclude整个页面，如@@||example.com^document以example.com开头的所有请求都不拦截。

2. 匹配地址的开头和结尾’|’

​ 地址前开头和结尾默认可以省略 号。如ad和``` ad* ```。

​ ‘|’放在前面匹配开头，放在后面匹配结尾。
​ 例如：
​ swf|可以匹配到http://example.com/annoyingflash.swf，但是不匹配http://example.com/swf/index.html。
​ |http://baddomain.example/可以匹配http://baddomain.example/banner.gif，但不匹配http://gooddomain.example/analyze?http://baddomain.example。

​ ‘||’标记用于匹配域名的开头。
​ ||example.com/banner.gif可以拦截

3. 分割符^

​ ^可以替代任何一个非字母，非数字，非’_’，非’-‘，非’%’，非’.’的任意一个字符。

高级用法

1. 设置过滤参数

​ 它决定了过滤的元素类型。

​ ‘$’开头各个参数以’,’分隔。

1.1可选参数：

  • script：通过HTML script标签加载的外部scripts。

  • image：regular images，典型的是通过HTML的img标签加载的图像。

  • stylesheet：外部的CSS文件

  • xmlhttprequest：使用XMLHttpRequest对象或fetch() API的生成的请求

  • subdocument：嵌入的页面，如iframe标签

  • ping：由navigator.sendBeacon()生成的请求。

  • websocket：通过WebSocket对象初始化的请求

  • webrtc：

  • document：the page itself, but only for exception rules. 使用此参数可以将整个iframe或website添加到白名单

  • elemhide：for exception rules only. Turn of element hiding rules on the page rather than all filter rules.

  • generichide：for exception only. turns off generic element hiding rules on the page.

  • genericblock：for exception only. turns off generic blocking rules.

  • popup：在新页面打开的网页。只有使用$popup参数被指定时，filters才默认不阻止弹出窗口。

  • font：外部字体文件。

  • media：诸如音乐，视频的regular media files

  • match-case：大小写敏感。

    */BannerAd.gif$match-case会block http://example.com/BannerAd.gif但unblock http://example.com/bannerad.gif

  • other：除了上述之外的参数类型。

1.2 参数取反：

​ 指定过滤规则不应用具体的参数。如~script, ~image, ~stylesheet, ~object, ~xmlhttprequest, ~subdocument, ~ping, ~websocket, ~webrtc, ~document, ~elemhide, ~other.

1.3 domain restrictions

domain=example.com | example.net. 过滤规则只应用于来自example.com或者example.net。

domain=~example.com. 过滤规则应用于除了example.com之外的域名。
domain=example.com | ~foo.example.com. 过滤规则应用于example.com但foo.example.com除外。

1.4 Sitekey restrictions

sitekey=abcdsitekeydbca | bcdesitekeyedcb. 过滤规则只应用于能够提供abcdsitekeydbca或bcdesitekeyedcb秘钥的网页。

1.5 Content Security Policies

csp=script-src: ‘none’. 拦截所有的script。

1.6 Redirecting requests to internal resources

参数：rewrite=
用于重写internal resource或者redirecting requests的URLs，它可以让internal resource或者redirecting requests失效而不报错。
abp-resource指定internal resource.
如$rewrite=abp-resource:blank-js会发送一个empty JavaScript。

​ 可用的internal resources有：

  • blank-text - empty text
  • blank-css - empty stylesheet
  • blank-js - empty JavaScript
  • blank-html - empty HTML document
  • blank-mp3 - a 0.1s silent MP3 audio file
  • 1x1-transparent-gif - 1x1 pixel transparent GIF image
  • 2x2-transparent-png - 2x2 pixels transparent PNG image
  • 3x2-transparent-png - 3x2 pixels transparent PNG image
  • 32x32-transparent-png - 32x32 pixels transparent PNG image

【注意】使用rewrite=参数其过滤规则必须是*或者以||作为开头。

2. 使用正则表达式

​ 如/banner\d+匹配banner123和banner321但不匹配banners。使用正则表达式会影响性能。

3. 特殊的注释

​ 这些注释只会作用于下载的过滤规则列表(filter lists)，而不会影响自定义的规则。他们会为过滤规则列表设置一些参数。

  • ! Homepage: http://example.com/ This comment determines which webpage should be linked as filter list homepage.
  • ! Title: FooList 为filter list设置title，一旦设置不可更名。
  • ! Expries: 5days 设置filter list的更新间隔时间。范围为1hour — 14days. 这个时间不是说到了就会立即更新，它受服务器的负载影响。
  • ! Redirect: http://example.com/list.txt filter list已被移动到新的下载地址上。若新下载地址与当前地址相同，则忽略此注释。
  • ! Version: 1234 filter list的版本。

Content Filters

Unfortunately, there are some advertisements that can’t be blocked at download time. This is because they’re embedded as text in the webpage itself. For that case, there are filters that run directly on the content of the page. They are split between two main types: element hiding and snippets.

element hiding filters指定页面中那个元素被隐藏。
snippet filters以代码片段的形式运行在页面上，阻止复杂的广告内容。

1.Basic rules

comtent filters的格式：<domains><separator><body>。

  • domains：一个或多个用都好隔开的域名。
  • separator：domains结束和Content filter开始的标志，它决定filter的类型。
  • body：filter的主体，它依赖filter的类型。
separator type body Content
## Element hiding CSS selector(domains may be empty)
#?# Element hiding emulation Extended CSS selector
#@# Element hiding exception Selector
#$# Snippet filter Snippet

2.Element hiding basic rules

​ 当把网页的源码下载后，可以选择性地隐藏某一部分，如广告展示的部分。例如：

​ 规则##.textad隐藏class属性为textad的element。
​ 规则###sponsorad隐藏id为sponsorad的element。
​ 规则##textad隐藏标签中的内容。

3.Limiting rules to certain domains

​ 只对某些域名下的网页进行hiding element。例如：
​ 规则：example.com##.sponsor可以作用在http://something.example.com/上，但不会作用在http://example.org/上。
​ 可指定多个域名，之间用逗号分隔，如：
​ domain1.example, domain2.example, domain3.example##.sponsor
​ ‘~’+domain表示不作用于该domain下的element，例如
​ ~example.com##.sponsor作用与除了exapmle.com之外的所有域名。example.com,~foo.example.com##.sponsor作用于example.com，但其子域foo.example.com除外。

【注意】这种限制只能写full domain names，不能只写一部分，也不能use domain as a replacement for domain.example, domain.test。

4.Attribute selectors

​ 根据属性值选择要隐藏的属性。例如：
​ ##table[width="80%"]会隐藏width属性值为80%的table标签。
​ ##div[title*="adv"]会隐藏所有title值里面包含adv的div标签。
​ ##div[title^="adv"][title$="ert"]会隐藏title值以adv开始并且以ert结束的div标签。
​ table[width="80%"][bgcolor="white"]会隐藏width=80%，bgcolor=white的table标签。

5.Extended CSS selectors

​ 使用extended CSS selectors会有性能影响。Extended CSS selector除了CSS中的选择器外，还有Adblock Plus自定义的CSS 选择器：

  • :-abp-has(selector)

    :-abp-has(>div>a.advertiser)：选择孩子标签是div，并且该div含有a标签，并且a标签有一个class属性为advertiser的标签。selector也可以再使用:-abp-has()。selector也可以是CSS selector。

    example.com#?#:-abp-has(.sponsored)会隐藏所有的页面，因为页面的<body>中的某个标签含有这个class属性，使用’>’或者’+’可以避免这种情况。

    eyeo.com#?#:-abp-has(code)会隐藏https://help.eyeo.com/en/adblockplus/how-to-write-filters页面中所有的元素。这是因为`标签中包含标签，而eyeo.com#?#:-abp-has(> code)只会隐藏`的父元素，不加’>’会隐藏整个页面。

  • :-abp-properties(properties)

    根据stylesheet properties选择要隐藏的element。
    :-abp-properties(width: 300px; height: 250px; )会选择stylesheet设置width为300px，height为250px的element。
    :-abp-properties(width: *px; height: 250px; )会选择stylesheet设置height为250px，width以px为单位的任意值的element。‘*’代表任意值。
    :-abp-properties(/width: 30[2-8]px; height: 250px; / )会选择width值在302-308px之间，且 height为250px的element。’//‘使用正则表达式。

    【注意】background-color的值使用rgb()获得。
    :-abp-properties(background-color: rgb(61, 156, 79))代替:-abp-properties(background-color: #3D9C4F)。

    :-abp-properties(content: 'Advertisement')会选择::before或::after的元素中内容为’Advertisement’的元素。::before，::after选择器

  • :-abp-contains(text)
    基于文本内容选择element。例如：
    div. sidebar > span :-abp-contents(Advertisement)会选择innerHTML为Advertisement，且class为.sidebar的div标签。它等同于div.sidebar > div :-abp-has(span:-abp-contains(Advertisement))。

    若设置eyeo.com#?#:-abp-contains(filters)，则https://help.eyeo.com/en/adblockplus/how-to-write-filters页面中不会隐藏任何element。若改为`eyeo.com#?#div:-abp-contains(filters)`，则包含filters字符串的div标签（还是包裹该div的标签我没弄清楚）将被隐藏（原文：`div.outer` (which contains the middle section of the page) is blocked because somewhere within the <body> is the word “filters”.）。eyeo.com#?#.article-heading:-abp-contains(filters)会选择class属性值为.article-heading的标签。

6.Exception rules

​ 作用在example.com上的##.texted可以用example.com#@#.texted取消。
​ 使用规则##aside.info作用在https://help.eyeo.com/en/adblockplus/how-to-write-filters上，`eyeo.com#@#aside`不会Exception anything。如果使用##aside规则，eyeo.com#@#aside.info同样不会Exception anything。规律规则必须严格相同，才会有效，如##aside.info canceled with eyeo.com#@#aside.info。

7.Snippet filters

​ snippet filters允许使用JavaScript代码来编辑。

​ 出于安全性的考虑，snippet filters仅被用在用户自定义的filters list中或者ABP Anti-Circumvention Filter List中。
