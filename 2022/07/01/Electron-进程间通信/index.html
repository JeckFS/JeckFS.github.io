Electron-进程间通信

发表于2022-07-01|更新于2022-07-01|Electron
|阅读量:

这里主要说的是主进程和渲染进程之间的通信方式。一般用在 从 UI 调用原生 API 或 从原生菜单触发 Web 内容的更改 的场景中。

1. 模式1: 渲染进程到主进程（单向）

渲染进程使用 ipcRenderer.send API 发送消息，主进程使用 ipcMain.on API 接收。

2. 模式2: 渲染进程到主进程（双向）

应用场景：从渲染器进程代码调用主进程模块并等待结果。

主进程通过 ipcMain.handle 接受请求并返回结果，渲染进程通过 ipcRenderer.invoke 发送请求并异步或者同步等待结果。

示例：从渲染器进程打开一个原生的文件对话框，并返回所选文件的路径。

3. 模式3: 主进程到渲染进程

将消息从主进程发送到渲染器进程时，需要指定是哪一个渲染器接收消息。 消息需要通过其 WebContents 实例发送到渲染器进程。 此 WebContents 实例包含一个 send 方法，其使用方式与 ipcRenderer.send 相同。

主进程通过 BrowserWindow.webContents.send 触发渲染进程的事件；
渲染进程通过 ipcRenderer.on 监听主进程触发的事件；

渲染进程通过 event.sender.send 触发主进程的事件；
主进程通过 ipcMain.on 监听渲染进程触发的事件；

4. 渲染进程到渲染进程

  • 方式一：将主进程作为渲染器之间的消息代理。 这需要将消息从一个渲染器发送到主进程，然后主进程将消息转发到另一个渲染器。
  • 方式二：从主进程将一个 MessagePort 传递到两个渲染器。 这将允许在初始设置后渲染器之间直接进行通信。
