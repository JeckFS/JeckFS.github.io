  • class 其实就是构造函数 constructor ，其中的方法存在与 Class.prototype 中。
  • class 中的函数没有分隔符 ,

使用 class 定义的 User 与用函数定义的 User 的区别：

  • 通过 class 创建的函数具有特殊的内部属性标记 [[IsClassConstructor]]: true，因此只能使用 new 关键字来创建。
  • 类方法不可枚举。 类定义将 “prototype” 中的所有方法的 enumerable 标志设置为 false。对一个对象调用 for..in 方法，不会遍历到方法。
  • 类总是使用 use strict。 在类构造中的所有代码都将自动进入严格模式。

2.类继承

易混淆点：
class.prototype 指的是 class 的定义， class.prototype.[[prototype]] 才是 class 的原型。

类继承示意图：

extends + 任意表达式：

子类重写父类方法，并调用父类方法：

  • 只能在子类的构造函数中调用父类构造函数，调用方式：super(...args)
  • 调用父类方法：super.method()
  • 不管是调用父类方法还是父类构造函数，super 都不用出现在首句。
  • 特殊地，在调用父类构造函数时，super 一定要在使用 this 之前调用。并且通过重写父类构造函数中的属性时，只能通过 super 进行赋值，不能直接赋值。

类字段重写的特殊情况：

类字段的初始化顺序：

  • 对于基类（还未继承任何东西的那种），在构造函数调用前初始化。
  • 对于派生类，在 super() 后立刻初始化。

现象：父类构造器总是会使用它自己字段的值，而不是被重写的那一个。
原因：new Rabbit() 调用了 super()，因此它执行了父类构造器，并且（根据派生类规则）只有在此之后，它的类字段才被初始化。在父类构造器被执行的时候，Rabbit 还没有自己的类字段，这就是为什么 Animal 类字段被使用了。
