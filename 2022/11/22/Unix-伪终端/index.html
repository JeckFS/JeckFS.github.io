Unix-伪终端

伪终端的典型使用场景

1. 典型应用一

适用于 FreeBSD、Mac OS X、Linux。

  • 基本实现方式：
    • 父进程打开 pseudo terminal master，然后调用 fork 创建子进程。
    • 子进程创建新会话；
      打开对应的 pseudo terminal slave；
      复制标准输入、标准输出、标准错误文件描述符；
      最后调用 exec()；
      至此 pseudo terminal slave 变为子进程的控制终端。
  • 对于子进程的标准输入、标准输出、标准错误来说，像是一个终端设备，该进程可以使用所有的终端 IO 函数，但是 slave 并不是一个真正的终端设备，因此对终端 IO 函数的调用都将被忽略。
  • 对于 pseudo terminal master 和 slave 来说，对任何一放的写入等同于对另一方的输入。

2. 典型应用二：Network Login Servers

这种场景下，伪终端内建于服务端以提供网络登陆的功能。如 telnetd 服务端以及 rlogind 服务端。

rlogind server 需要使用 IO 多路复用技术或者两个进程/线程同时读写 TCP/IP 流以及与 PTY master 交互的数据流。

3. 典型应用三：Windowing System Terminal Emulation

窗口系统通常提供一个终端模拟器。通过终端模拟器，可以在命令行环境中通过 shell 运行程序。每个 shell 在自己的窗口中执行。
终端模拟器需要根据它所模拟的设备类型来响应返回码。这些返回码可以通过 man termcap; man terminfo; 进行查看。

当用户改变 terminal emulator window 的窗口大小时，terminal emulator 会在伪终端的 master 一侧上执行 TIOCSWINSZ ioctl 命令，进而改变 slave 一侧的窗口大小。
若心的窗口大小与当前不同，内核会发送 SIGWINCH 信号给 PTY slave 一侧的前台进程组。当窗口大小改变时，若应用需要重绘屏幕，应用可以捕捉到 SIGWINCH 信号，并发起 TIOCGWINSZ ioctl 命令去获取新的屏幕大小，然后重绘屏幕。

4. 典型应用四：script Program

script 程序将终端会话期间所有输入和输出信息复制到一个文件中。为完成此工作，该程序将自己置于终端和一个新调用的登陆 shell 之间。

script 程序通常是从登陆 shell 启动，并且 shell 还要等待 script 程序的终止。

script 程序运行时，位于 PTY 从设备上的终端行规程的所有输出都将复制到脚本文件中（通常是 typescript）。因为击键通常由该行规程模块回显，所以该脚本文件也包括了输入的内容。但是不会包含键入的口令。

5. 典型应用五：Running Coprocesses

在协同进程中，不能使用标准 IO 哭进行输入、输出时，标准 IO 库会完全缓冲标准输入和标准输出，从而引起死锁。
可以将伪终端置于两个进程之间，诱使协同进程认为它是有终端驱动的（标准 IO 库会将这两个流设置为行缓冲），而非另一个进程。

父进程在自身和协同进程之间获得伪终端的方法：

  • 父进程直接调用 pty_fork() 而不是调用 fork

打开伪终端设备

当打开 PTY 设备文件时，应用不需要设置 O_TTY_INIT 标志，因为首次打开时，在PTY 设备的 slave 一侧，任何所需的非标准化的 termios 标志都会自动设置。

参数 oflag 与 open() 中的标志参数相同，是一个位掩码，作用是指定 master 设备如何打开。并不是所有的标志都支持，所支持的标志如下：

  • O_RDWR：打开 master 设备进行读写
  • O_NOCTTY：阻止 master 设备变为调用者的控制终端。

在 slave 伪终端设备使用前，其权限需要使用 grantpt() 设置。该函数将 slave 节点的用户 ID 设置为调用者的实际用户 ID，组 ID 被设置为 unspecified 值。对于 individual owners，权限为读和写；对于组拥有者（0620）拥有写权限。

实现中通常将 PTY slave 设备的组所有者设置为 tty 组。需要写入到所有 active 终端的权限 具有写入到所有 active 终端权限的应用对于 tty 组来说都是 set-group-ID 。常见的程序如 wall(1) 和 write(1)。

为了更改 slave 设备的权限， grantpt 函数需要 fork 并 exec 一个 set-user-ID 程序（如在 Solaris 中是 /usr/lib/pt_chmod）。因此，若调用者捕捉 SIGCHLD 信号则其行为是未定义的。

unlockpt() 用来准许对为终端 slave 设备的访问，从而允许应用打开该设备。

注意：在 grantpt() 与 unlockpt() 中，文件描述符参数是与伪终端 master 设备相关联的文件描述符。

在给定 master 的文件描述符的情况下，ptsname 函数用于查找伪终端 slave 设备的路径名。注意：返回的路径名会存储在静态存储区，后续的调用会覆盖该部分内容。

ptym_open 函数用于打开下一个可用的 PTY master 设备。调用者需要为参数 pts_name 分配一个数组以存储 slave 设备名。若调用成功，参数 pts_name 会存储 slave 设备名。

ptys_open 函数用于打开对应的 slave 设备。参数 pts_name 是 ptym_open 函数获取的 slave 设备名。

通常，这两个函数不会直接被调用，取而代之的是，pty_fork 函数会调用它们，然后 fork 一个子进程。

这两个函数一个简单应用：
在调用 pty_fork 函数时，通常主进程调用 ptym_open 来打开 master 并获取 slave 的设备名。然后主进程 fork 出子进程，子进程在调用 setsid 函数建立一个新会话后调用 ptys_open 函数打开 slave。这也是为什么 slave 变为子进程的控制终端的原因。

pty_fork 函数

参数 ptrfdm：获取返回的 PTY master 设备的文件描述符。
参数 slave_name：用于获取 slave 的设备名。其存储空间需调用者来分配。
参数 slave_termios：若非空，系统将使用该引用初始化 slave 设备的终端行规程。若为空，则 slave 设备的 termois 结构体将被设置为默认值。
参数 slave_winsize：若非空，则用于初始化 slave 的窗口大小。若为空， winsize 结构体通常初始化为 0.

pty 程序

pty 程序将在自己创建的（且已连接伪终端的） session 中执行其他程序。

关于 pty 程序的一点说明：

虽然子进程在执行 exec 程序时先于父进程运行，但其输出并不会因此而丢失。
