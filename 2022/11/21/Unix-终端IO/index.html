Unix-终端IO

发表于2022-11-21|更新于2022-12-09|Unix
|阅读量:

综述

终端 IO 的工作模式：

  • 规范模式输入处理（默认模式）
    对终端输入以行为单位进行处理。
  • 非规范模式输入处理
    输入字符不装配成行。如 vi 编辑器。

POSIX.1 定义了 11 个特殊输入字符，其中 9 个可更改。如 Ctrl+D 是文件结束符，Ctrl+Z 是挂起字符。

  • 终端设备是由通常位于内核中的终端驱动程序控制的。每个终端设备都有一个输入队列和输出队列。
  • 若打开了回显功能，则在输入队列和输出队列之间有一个隐含的连接。
  • 输入队列长度限制：MAX_INPUT。当一个特定设备的输入队列已经填满时，系统的行为依赖于实现，大多数 UNIX 系统回显响铃字符。
  • 输入行的最大字节数限制：MAX_CANON
  • 输出队列的长度也是由限制的，但程序并不能获取。输出队列将要填满时，内核会使写进程休眠，直至写队列中有可用空间。

termios.h 头文件

所有可以检测和更改的终端设备特性都包含在 termios 机构中，该结构定义在头文件 termios.h 中。

  • 输入标志 c_iflag：通过终端设备驱动程序控制字符的输入。如剥除输入字节的第 8 位、允许输入奇偶校验等。
  • 输出标志 c_oflag：控制驱动程序输出。如执行输入处理、将换行符转换为 CR/LF 等。
  • 控制标志 c_cflag：影响 RS-232 串行线。如忽略调制解调器的状态线、每个字符的一个或两个停止位。
  • 本地标志 c_lflag：影响驱动程序和用户之间的接口。如回显打开或关闭、可视地擦除字符、允许终端产生的信号以及对后台输出的作业控制停止信号。
  • 控制字符数组 c_cc：包含了所有可以更改的特殊字符。cc_t 类型的长度足以保存每个特殊字符，典型的时 unsigned char。
  • 类型 tfflag_t 的长度足以保存每个标志值，常被定义为 unsigned int 或 unsigned long

特殊输入字符

POSIX.1 要求换行符和回车符（分别是 \n 和 \r）或者 STOP(Ctrl+S) 和 START(Ctrl+Q) 字符（依赖于实现）。

通过更改 termios 结构中国的 c_cc 数组的相应项可以更改特殊字符的值，其中数组的下标以字母 V 开头。

c_cc 数组中某项设置为 _POSIX_VDISABLE 表示禁止使用相应特殊字符。

程序说明：

  • 禁用中断键与忽略中断信号时不同的。本程序只是禁用使终端驱动程序产生 SIGINT 信号的特殊字符，我们仍旧可以使用 kill 函数将该信号发送至进程。

大多数特殊字符在被终端驱动程序识别并进行特殊处理后会被丢弃，并不将它们返回给执行读终端操作的进程。返回给读进程的例外字符是换行符（NL、EOL、EOL2）和回车符（CR）。

BREAK 字符：BREAK 实际上并不是一个字符而是在异步串行数据传送时发生的一个条件。根据串行接口的不同，可有多种方式通知设备驱动程序发生了 BREAK 条件。
对于异步串行数据传送，BREAK 是一个 0 值的位序列，其持续时间长于要求发送一个字节的时间。整个 0 值位序列被视为一个 BREAK。BREAK 字符可由 tcsendbreak 函数发送。

opt 参数说明：

tcsetattr 函数的返回状态易混淆，即使未能执行所有要求的动作，它也返回 OK，当返回 OK 时，仍需调用 tcgetattr 检查是否属性设置成功。

终端选项标志

列出的所有选项标志（除屏蔽字标志外）都用一位或多位（设置或清除）表示。

屏蔽字表示定义多个位，它们组合在一起，可以定义一组值。屏蔽字标志有一个定义名，每个值也有一个名字。例如，为了设置字符长度，首先用字符长度屏蔽字标志 CSIZE 将表示字符长度的位清 0，然后设置下列值之一：CS5, CS6, CS7, CS8。

stty 命令

stty 的作用等同于 tcgetattr 和 tcsetattr 函数，用于检查和更改终端选项。

波特率函数

波特率单位是 bit / s （bit per second），大多数终端设备的输入和输出使用同一波特率，若硬件支持，可以将他们设置为不同值。

输入、输出波特率是存储在设备的 termios 结构中的。更改时需调用 tcsetattr 函数。

行控制函数

4 个函数中的 fd 参数要求引用一个终端设备，否则出错返回 -1，errno 设置为 ENOTTY。

  • tcdrain：等待所有输出都被传递。

  • tcflow：用于对输入和输出流控制进行控制
    |action 参数取值|描述|
    |:—-:|:—-:|
    |TCOOFF|输出被挂起|
    |TCOON|重新启动以前被挂起的输出|
    |TCIOFF|系统发送一个 STOP 字符，这将使终端设备停止发送数据|
    |TCION|系统发送一个 START 字符，这将使终端设备恢复发送数据|

  • tcflush：冲洗（抛弃）输入缓冲区（其中的数据是终端驱动程序已接收到，但用户程序尚未读取的）或输出缓冲区（其中的数据是用户程序已经写入，但尚未被传递的）。
    |queue 参数取值|描述|
    |:—-:|:—-:|
    |TCIFLUSH|冲洗输入队列|
    |TCOFLUSH|冲洗输出队列|
    |TCIOFLUSH|冲洗输入队列和输出队列|

  • tcsendbreak：在指定的时间区间内发送连续的 0 值位流。若 duration 参数为 0，则此中种传递延续 0.25 ～ 0.5s。若非 0，则传递时间依赖于实现。
